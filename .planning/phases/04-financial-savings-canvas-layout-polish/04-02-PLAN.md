**Canvas panel options structure:**

The Canvas panel in Grafana 12 uses `options.root` to define elements. Each element has:
- `name`: identifier
- `type`: "rectangle" for colored blocks, "text" for labels
- `placement`: absolute positioning {top, left, width, height}
- `background.color`: data-bound or fixed
- `config`: display options

**CRITICAL: Canvas panel JSON structure for Grafana 12.4.1:**

```json
"options": {
  "inlineEditing": true,
  "showAdvancedTypes": false,
  "root": {
    "elements": [
      {
        "name": "west_pv2",
        "type": 1,
        "config": {
          "align": "center",
          "valign": "middle",
          "text": { "fixed": "West PV2" },
          "color": { "fixed": "white" },
          "size": 14
        },
        "background": {
          "color": { "field": "West PV2", "fixed": "#1a1a2e" }
        },
        "border": {
          "color": { "fixed": "#333" },
          "width": 1
        },
        "placement": {
          "top": 20,
          "left": 50,
          "width": 150,
          "height": 100
        },
        "constraint": {
          "horizontal": "left",
          "vertical": "top"
        }
```

**Actually, for a proper heatmap showing day-of-week vs hour-of-day (calendar style), or daily totals over time:**

Per D-61, the heatmap shows daily production patterns. The best approach for a Grafana heatmap panel:

**Option A — Time-series heatmap (hour of day vs date):**
Shows production intensity by hour across days. Each cell = 1 hour of 1 day. This is the most useful "calendar heatmap" pattern in Grafana's native heatmap.

Query:
```sql
SELECT
  DATE_BIN(INTERVAL '1 hour', time, '1970-01-01T00:00:00Z') AS time,
  AVG(
    COALESCE(pv1_power_sensor, 0) + COALESCE(pv2_power_sensor, 0) +
    COALESCE(pv3_power_sensor, 0) + COALESCE(pv4_power_sensor, 0)
  ) / 1000.0 AS production_kw
FROM "East Microinverter"
WHERE time >= now() - INTERVAL '90 days'
GROUP BY 1
ORDER BY 1
```

This gives hourly power values. The Grafana heatmap panel with `calculate: true` and yAxis bucket size of 1 hour will create a heat grid of date (x) × hour-of-day (y).

**However**, Grafana's heatmap panel needs a specific data format. For a "daily total" heatmap (one value per day, no hour dimension), use:

```sql
SELECT
  date_trunc('day', time) AS time,
  SUM(hourly_kwh) AS daily_kwh
FROM (
  SELECT
    DATE_BIN(INTERVAL '1 hour', time, '1970-01-01T00:00:00Z') AS bucket,
    time,
    AVG(COALESCE(pv1_power_sensor, 0) + COALESCE(pv2_power_sensor, 0) + COALESCE(pv3_power_sensor, 0) + COALESCE(pv4_power_sensor, 0)) / 1000.0 AS hourly_kwh
  FROM "East Microinverter"
  WHERE time >= now() - INTERVAL '90 days'
  GROUP BY 1, time
)
GROUP BY 1
ORDER BY 1
```

If subqueries fail, simplify to daily avg power × solar hours:
```sql
SELECT
  date_trunc('day', time) AS time,
  AVG(COALESCE(pv1_power_sensor, 0) + COALESCE(pv2_power_sensor, 0) + COALESCE(pv3_power_sensor, 0) + COALESCE(pv4_power_sensor, 0)) / 1000.0 * 10 AS daily_kwh_est
FROM "East Microinverter"
WHERE time >= now() - INTERVAL '90 days'
GROUP BY 1
ORDER BY 1
```
