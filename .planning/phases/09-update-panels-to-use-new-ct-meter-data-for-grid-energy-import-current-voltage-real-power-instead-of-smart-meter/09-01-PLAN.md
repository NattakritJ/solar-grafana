<objective>
Migrate 12 panels from Smart Meter to the new CT grid meter (`grid` table) for grid power, voltage, and current data — leaving frequency and power_factor panels on Smart Meter since those fields don't exist in the CT table.

Purpose: The `grid` CT meter placed at the grid connection point provides more accurate real-time measurement of grid import power, voltage, and current. The signed convention (negative = backfeed) also improves backfeed detection accuracy.

Output: `solar-pv-monitor.json` with 10 targeted SQL string replacements covering all 12 affected panels. Dashboard version bumped. No panel additions, no gridPos changes — pure SQL field/table substitution.
</objective>

## CT `grid` Table Schema (Confirmed Live)
Fields: `power` (W, signed: positive=import, negative=backfeed), `current` (A, signed), `voltage` (V, always positive), `energy_today` (kWh), `energy_backfeed_today` (kWh), `time`
No `_sensor` suffix. No tag columns. Data since 2026-04-03T17:21 UTC.

## Panels That STAY on Smart Meter (DO NOT CHANGE)
- Panel 17 (Grid Frequency) — `frequency_sensor` (CT has no frequency)
- Panel 18 (Power Factor) — `power_factor_sensor` (CT has no power_factor)
- Panel 20 refId=B (Frequency series) — stays on Smart Meter
- Panel 21 refId=A (Power Factor series) — stays on Smart Meter
- Panel 801 refId=Load — GREATEST in subquery is for Load calc, stays on Smart Meter

## Current SQL → New SQL Mapping

### Change 1 — Panels 4 & 11 (2 occurrences, identical SQL)
OLD (exact JSON string, 2 occurrences):
`SELECT power_sensor AS grid_kw FROM \"Smart Meter\" WHERE time >= now() - INTERVAL '2 minutes' ORDER BY time DESC LIMIT 1`

NEW:
`SELECT power AS grid_kw FROM \"grid\" WHERE time >= now() - INTERVAL '2 minutes' ORDER BY time DESC LIMIT 1`

### Change 2 — Panels 6 & 905 (2 occurrences, identical SQL)
OLD (exact JSON string, 2 occurrences):
`SELECT COALESCE(selector_last(power_sensor, time)['value'], 0) AS grid_w FROM \"Smart Meter\" WHERE time >= now() - INTERVAL '2 minutes'`

NEW:
`SELECT COALESCE(selector_last(power, time)['value'], 0) AS grid_w FROM \"grid\" WHERE time >= now() - INTERVAL '2 minutes'`

### Change 3 — Panel 801 Grid series ONLY (1 occurrence — the "Grid" alias line, NOT the "Load" GREATEST)
OLD (exact JSON string, 1 occurrence):
`GREATEST(AVG(power_sensor), 0) / 1000.0 AS \"Grid\" FROM \"Smart Meter\" WHERE time >= $__timeFrom AND time <= $__timeTo GROUP BY 1 ORDER BY 1`

NEW (remove GREATEST, allow negative backfeed, switch table):
`AVG(power) / 1000.0 AS \"Grid\" FROM \"grid\" WHERE time >= $__timeFrom AND time <= $__timeTo GROUP BY 1 ORDER BY 1`

⚠️ CAUTION: There is a second GREATEST on the Load series — `GREATEST(AVG(power_sensor), 0) AS pw FROM \"Smart Meter\"` — do NOT touch that one. Only replace the one ending with `AS \"Grid\" FROM \"Smart Meter\"`.

### Change 4 — Panel 16 (1 occurrence)
OLD: `SELECT voltage_sensor AS voltage FROM \"Smart Meter\" WHERE time >= now() - INTERVAL '2 minutes' ORDER BY time DESC LIMIT 1`
NEW: `SELECT voltage AS voltage FROM \"grid\" WHERE time >= now() - INTERVAL '2 minutes' ORDER BY time DESC LIMIT 1`

### Change 5 — Panel 19 (1 occurrence)
OLD: `SELECT current_sensor AS current FROM \"Smart Meter\" WHERE time >= now() - INTERVAL '2 minutes' ORDER BY time DESC LIMIT 1`
NEW: `SELECT current AS current FROM \"grid\" WHERE time >= now() - INTERVAL '2 minutes' ORDER BY time DESC LIMIT 1`

### Change 6 — Panel 20 Voltage series (1 occurrence — refId=A only)
OLD: `SELECT DATE_BIN(INTERVAL '5 minutes', time) AS time, AVG(voltage_sensor) AS \"Voltage\" FROM \"Smart Meter\" WHERE time >= $__timeFrom AND time <= $__timeTo GROUP BY 1 ORDER BY 1`
NEW: `SELECT DATE_BIN(INTERVAL '5 minutes', time) AS time, AVG(voltage) AS \"Voltage\" FROM \"grid\" WHERE time >= $__timeFrom AND time <= $__timeTo GROUP BY 1 ORDER BY 1`

⚠️ CAUTION: refId=B (Frequency series) stays on Smart Meter. Only replace the Voltage series.

### Change 7 — Panel 21 Current series (1 occurrence — refId=B only)
OLD: `SELECT DATE_BIN(INTERVAL '5 minutes', time) AS time, AVG(current_sensor) AS \"Current\" FROM \"Smart Meter\" WHERE time >= $__timeFrom AND time <= $__timeTo GROUP BY 1 ORDER BY 1`
NEW: `SELECT DATE_BIN(INTERVAL '5 minutes', time) AS time, AVG(current) AS \"Current\" FROM \"grid\" WHERE time >= $__timeFrom AND time <= $__timeTo GROUP BY 1 ORDER BY 1`

⚠️ CAUTION: refId=A (Power Factor series) stays on Smart Meter. Only replace the Current series.

### Change 8 — Panel 33 (Today's Backfeed Count) (1 occurrence)
OLD rawSql (with \n newlines in JSON):
`SELECT COUNT(*) AS \"Count\"\nFROM \"Smart Meter\"\nWHERE time >= date_trunc('day', now() AT TIME ZONE 'Asia/Bangkok') AT TIME ZONE 'UTC'\n  AND power_sensor < 0`

NEW:
`SELECT COUNT(*) AS \"Count\"\nFROM \"grid\"\nWHERE time >= date_trunc('day', now() AT TIME ZONE 'Asia/Bangkok') AT TIME ZONE 'UTC'\n  AND power < 0`

### Change 9 — Panel 34 (Max Backfeed Power) (1 occurrence)
OLD rawSql:
`SELECT ABS(MIN(power_sensor)) AS \"Max Export\"\nFROM \"Smart Meter\"\nWHERE time >= date_trunc('day', now() AT TIME ZONE 'Asia/Bangkok') AT TIME ZONE 'UTC'\n  AND power_sensor < 0`

NEW:
`SELECT ABS(MIN(power)) AS \"Max Export\"\nFROM \"grid\"\nWHERE time >= date_trunc('day', now() AT TIME ZONE 'Asia/Bangkok') AT TIME ZONE 'UTC'\n  AND power < 0`

### Change 10 — Panel 35 (Grid Backfeed Log) (1 occurrence)
OLD rawSql:
`SELECT\n  time AS \"Time\",\n  ABS(power_sensor) AS \"Export Power (W)\",\n  voltage_sensor AS \"Voltage (V)\",\n  current_sensor AS \"Current (A)\"\nFROM \"Smart Meter\"\nWHERE time >= $__timeFrom AND time <= $__timeTo\n  AND power_sensor < 0\nORDER BY time DESC\nLIMIT 100`

NEW:
`SELECT\n  time AS \"Time\",\n  ABS(power) AS \"Export Power (W)\",\n  voltage AS \"Voltage (V)\",\n  current AS \"Current (A)\"\nFROM \"grid\"\nWHERE time >= $__timeFrom AND time <= $__timeTo\n  AND power < 0\nORDER BY time DESC\nLIMIT 100`
