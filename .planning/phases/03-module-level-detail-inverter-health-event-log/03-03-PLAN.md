**Queries (3 — one per event source, merged via transformation per D-43):**

Query A (refId: "Backfeed"):
```sql
SELECT
  time AS "Time",
  'Smart Meter' AS "Source",
  'Backfeed' AS "Event",
  CAST(ABS(power_sensor) AS VARCHAR) || ' W' AS "Detail"
FROM "Smart Meter"
WHERE time >= $__timeFrom AND time <= $__timeTo
  AND power_sensor < 0
ORDER BY time DESC
LIMIT 50
```
Note: If InfluxDB 3 Core does not support CAST...AS VARCHAR concatenation, use a simpler approach:
```sql
SELECT
  time AS "Time",
  'Smart Meter' AS "Source",
  'Backfeed' AS "Event",
  ABS(power_sensor) AS "Detail"
FROM "Smart Meter"
WHERE time >= $__timeFrom AND time <= $__timeTo
  AND power_sensor < 0
ORDER BY time DESC
LIMIT 50
```
The Detail column will show the numeric power value. This is simpler and more reliable.

Query B (refId: "EastEvents"):
```sql
SELECT
  time AS "Time",
  'East Inverter' AS "Source",
  CASE
    WHEN alarm_sensor != 0 THEN 'Alarm'
    WHEN fault_sensor != 0 THEN 'Fault'
    ELSE 'State Change'
  END AS "Event",
  CASE
    WHEN alarm_sensor != 0 THEN alarm_sensor
    WHEN fault_sensor != 0 THEN fault_sensor
    ELSE device_state_sensor
  END AS "Detail"
FROM "East Microinverter"
WHERE time >= $__timeFrom AND time <= $__timeTo
  AND (alarm_sensor != 0 OR fault_sensor != 0 OR device_state_sensor != 2)
ORDER BY time DESC
LIMIT 50
```
Note: `device_state_sensor != 2` filters out normal "Producing" state — only log non-producing state transitions.
