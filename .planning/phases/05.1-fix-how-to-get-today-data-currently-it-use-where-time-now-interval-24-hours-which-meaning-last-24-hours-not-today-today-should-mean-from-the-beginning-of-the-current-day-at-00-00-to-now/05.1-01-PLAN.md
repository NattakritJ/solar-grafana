# Plan 05.1-01: Fix All 18 "Today" SQL Boundaries to Calendar-Day Expression

**Phase:** 05.1 — Fix "Today" data boundary  
**Plan:** 01 of 01  
**File:** `solar-pv-monitor.json`  
**Complexity:** Low — two global string replacements, zero structural changes

---

## Goal

Replace every rolling-window boundary (`now() - INTERVAL '24 hours'` and `now() - INTERVAL '1 day'`) used to express "today" with the correct calendar-day boundary for Bangkok timezone:

```
date_trunc('day', now() AT TIME ZONE 'Asia/Bangkok') AT TIME ZONE 'UTC'
```

After this plan all 18 affected `rawSql` fields will anchor "today" to 00:00 Bangkok time, not to a sliding 24-hour window.

---

## Pre-Conditions

- `solar-pv-monitor.json` exists at project root (5,451 lines)
- No other plan is in progress modifying `solar-pv-monitor.json`
